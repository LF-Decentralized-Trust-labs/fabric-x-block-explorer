## Branch Strategy
Each branch stacks on the previous one:
```
main ← feat/01-db-schema ← feat/02-db-writer ← feat/03-block-pipeline ← feat/04-main
```
When a PR merges to upstream `main`, rebase the next branch:
```bash
git checkout feat/02-db-writer
git rebase upstream/main
git push --force-with-lease
```
